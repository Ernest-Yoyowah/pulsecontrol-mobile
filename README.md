# PulseControl Mobile

> **Professional wireless MIDI surface for live performance and studio work**

Built by **Ernest Keyz Studios**

---

## Overview

PulseControl Mobile turns your iOS or Android device into a high-performance, wireless MIDI controller that communicates with the **PulseControl Bridge** desktop app over WebSocket. It provides 8 motorless faders and 8 rotary knobs, scene-based CC mapping, and a purpose-built landscape UI optimised for dark stage environments.

```
iPhone/iPad  ──WiFi──▶  PulseControl Bridge (macOS)  ──MIDI──▶  DAW / hardware
```

---

## Features

| Feature               | Details                                                                                      |
| --------------------- | -------------------------------------------------------------------------------------------- |
| **8 velocity faders** | Full-track pan gesture, snaps to touch point, per-CC assignment                              |
| **8 rotary knobs**    | Vertical-pan gesture, same CC pipeline                                                       |
| **Scene management**  | Unlimited named scenes, each stores independent CC maps                                      |
| **CC reassignment**   | Per-control CC editor (0–127) inside each scene                                              |
| **QR pairing**        | Scan a QR code from the Bridge app to set the WebSocket URL instantly                        |
| **Stage Dim Mode**    | 58 % dark overlay that makes the screen unreadable from FOH while keeping touch active       |
| **Haptic feedback**   | Optional impact haptics on gesture start/end (respects system setting)                       |
| **Auto-reconnect**    | Configurable — retries up to 5× at 3 s intervals, respects the toggle in Settings            |
| **Offline mode**      | "Continue offline" bypasses connection and keeps last loaded scene                           |
| **Hello handshake**   | On connect the app sends `{"type":"hello","client":"pulsecontrol-mobile","version":"1.0.0"}` |

## Getting Started

### Prerequisites

- Node.js 20+
- Expo CLI: `npm install -g expo-cli` (or use `npx expo`)
- iOS Simulator / physical device with Expo Go **or** Android emulator

### Install & Run

```bash
git clone git@github.com-personal:Ernest-Yoyowah/pulsecontrol-mobile.git
cd pulsecontrol-mobile
npm install
npx expo start
```

Press `i` for iOS simulator, `a` for Android emulator, or scan the QR code with Expo Go.

### Regenerate Icons

```bash
pip3 install Pillow
python3 scripts/generate-icons.py
```

## WebSocket Protocol

The app connects to `ws://<host>:8765` (default). All messages are JSON.

### App → Bridge

```jsonc
// Sent immediately on connect
{ "type": "hello", "client": "pulsecontrol-mobile", "version": "1.0.0" }

// Fader or knob moved
{
  "type": "cc",
  "channel": 1,      // MIDI channel 1–16 (from Settings)
  "cc": 1,           // CC number 0–127
  "value": 64        // 0–127
}
```

### Bridge → App (future)

```jsonc
{ "type": "ack" }
{ "type": "error", "message": "..." }
```

### CC Assignment Defaults

| Control   | Default CC |
| --------- | ---------- |
| Fader 1–8 | CC 1–8     |
| Knob 1–8  | CC 11–18   |

CC assignments are stored per-scene and editable in the Scenes tab.

## Connection Setup

1. Open the **PulseControl Bridge** desktop app on your Mac.
2. Note the WebSocket address shown (e.g. `ws://192.168.1.10:8765`).
3. On the mobile app, tap **Connect** (or navigate to the Connection screen).
4. Either:
   - **Type** the address manually, or
   - Tap **SCAN QR** and scan the QR code displayed by the Bridge app.
5. Tap **CONNECT** — the status dot turns green.

The Bridge and device must be on the **same Wi-Fi network**.

The desktop companion app lives at:

```
../pulsecontrol-bridge/
```

Stack: Tauri v2 · Rust · React · TypeScript · Tailwind CSS  
WebSocket server: tokio + axum on port 8765  
MIDI output: midir → CoreMIDI virtual port (macOS)  
Device discovery: mDNS / Bonjour
